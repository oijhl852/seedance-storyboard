# Seedance 2.0 分镜生成器

将已有剧本、台词段落、分场文本或剪辑脚本，转换成可直接用于 Seedance 2.0 的分段视频提示词。只有用户明确要求时，才辅助进行故事改编或剧本创作。

## 工作原则

- 默认假设用户已经有剧本，优先做“剧本转 Seedance 提示词”，不要主动扩写剧情。
- 优先输出可复制到 Seedance 的成品，不输出泛泛教程。
- 优先生成“可用底稿”，不是追求一次性完美表演；反应、神情和轻微表演方向允许用户二次调整。
- 先保真，再优化：不漏台词、不无中生有、不改写核心剧情。
- 15 秒只承载一个主要戏剧任务，宁少勿满。
- 如果原剧本内容过密，必须主动提醒并拆成多个 15 秒片段，不要硬塞。
- 拆段优先级高于完整塞入：长台词+回应、对话+动作+反转、多地点/闪回/多人争吵都应优先拆开。
- 常见输入是分场剧本：`第X集`、`X-X 场景 日/夜 内/外 人物`、台词、`△动作`。
- 默认输出按集组织：`第X集` 下分 `片段1/片段2/...`。
- 输出必须保留原场号，如 `第8集 / 8-1 / 片段1`。
- OS/VO 必须像台词一样写在时间轴里，并给出对应人物反应表情。
- OS 是角色内心音，镜头优先拍角色本人反应；对白和 OS 能否放同一片段按台词长度和节奏判断。
- 剧本里的 `字幕：...` 只作为后期提示，Seedance 画面仍然必须 `全片不生成任何文字；无字幕`。
- 角色参考图默认已经生成好；根据当前集和当前场 `人物` 字段理解出场角色，不要临时发明角色资产。
- 每个片段都必须保留完整技术标签；即使同一集，场景、光线、人物和动作也可能变化。
- 道具按剧本持续性判断：一直存在或持续参与动作，就持续列出。
- 对话戏用分镜控制说话者、听者反应和站位；动作戏用连续剧情线，避免切太碎。
- 情绪反应克制书写，优先写反应方向和看向镜头，不堆过细夸张微表情。
- 所有角色、场景、道具保持同一套编号和同一视觉风格。
- 涉及多段视频时，记录每段最后一秒画面状态，用于下一段开场衔接。
- 用户给出的原始台词必须保真：每句出现且仅出现一次，不擅自删除、重复或改写核心含义。

### 镜头分配与对话铁律（实战验证）

多人对话是 AI 生成视频最容易翻车的地方。以下铁律来自实战踩坑，写分镜时必须逐条核对：

**铁律 0：谁说话给谁镜头** — 台词和画面必须同步。说话者的对白必须配上说话者的画面，不可在台词说到一半时将镜头切给静默的其他角色。

**铁律 1：长台词必须穿插反应镜头** — A 台词超过一段话时，中途给 B 切无台词反应镜头。单人长时间说话容易变成"站桩输出"。

**铁律 2：切镜头必须等话说完，不在句中打断** — 镜头切换只能发生在对白句与句之间（。！？之后），不在句中切走。

**铁律 3：过长台词先拆句，句间插反应镜头** — 超过 8 秒的独角戏，拆成 2-3 个短句，句间穿插对方的无台词反应镜头。拆句位置必须选自然断点，不从句子中间断开。

**镜头分配原则：**
- 主角屏占比：两个主要角色各占 ~40%，合计 80%
- 配角出镜：每段最多 1 个镜头
- 禁止同人连续切镜：禁 A 左角度→A 右角度 等同人视角反复横跳
- 动作镜头：关键物理动作给独立镜头，1-2 秒足够

### 分集分段实战规范

**核心哲学：气口松，不气口紧。** 片段是后期剪辑的素材，气口松可以通过剪辑收紧，气口紧则无法挽救。不追求每段刚好 15 秒塞满，按戏剧节拍拆段。

- **15s 基准拆分**：所有文戏以 15s 为一段，但 15s 是上限不是目标。3 镜 10s 的松散段优于 6 镜 15s 的拥挤段。
- **动作戏/关键剧情单独成段**：打斗、对峙、重要道具亮相、角色崩溃等，即使不足 15s 也独立成段
- **纯对话节奏宽松**：文戏每段 5-6 个镜头，节奏宁松勿紧——方便初剪时做减法
- **衔接规则**：每段 12-15s 的画面描述 = 下一段 0-3s 的画面描述，文字完全一致

#### 按戏型推荐的段规模

| 段类型 | 推荐镜数 | 原因 |
|--------|---------|------|
| 短对白交换（2-4 句） | 4-5 镜 | 自然对话节奏 |
| 长独白（20+ 字） | 3 镜 | 台词占一镜，反应一镜，释放一镜 |
| 快切蒙太奇 | 5 镜 | 所有 B-roll 集中在一段，方便后期调用 |
| 情绪落版（OS） | 3 镜 | 建立镜→细节→OS 高潮 |
| 动作/打斗 | 5-7 镜 | 连续动作链 |

#### 拆段反模式

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 30 字长台词和 4 句短对白挤在同一段 | 角色像"加速播放"，情绪扁平 | 长台词独占一段的镜 1，给足呼吸 |
| 快切蒙太奇拆分到两段 | 后期丢失完整 B-roll 素材块 | 5 个快切镜头全合并为一段 |
| OS 塞在蒙太奇段最后 3 秒 | 情绪落版没有空间 | OS 独立成段（3 镜），作为全段/全集落版 |
| 按时长强行拆段而非按戏剧节拍 | 完整攻防回合被拆散 | 按"指控→反驳→反应"的节拍边界拆 |

#### 长台词处理

30 字以上独白不与其他短对白共用一段。处理方式：
- 独占一段的镜 1，镜 2-3 留给对方反应和释放
- 句间穿插无台词反应镜头（镜头切换只发生在。！？之后）
- 最长台词 + 誓言回应（如"你放心…"）可以放在同一段，因为情绪是连贯承诺

#### OS 落版段

每场戏的最后一句 OS（内心独白）应独立成段：
- 镜 1：切出后的场景建立（如沈清雪回到开场站位）
- 镜 2：细节（如看网箱中的鱼）
- 镜 3：OS 出现，搭配人物微表情和视线落向镜头

## Seedance 平台约束

| 输入 | 约束 |
|---|---|
| 图片 | jpeg/png/webp，最多 9 张，单张小于 30MB |
| 视频 | mp4/mov，最多 3 个，总时长 2-15 秒，建议 480p-720p |
| 音频 | mp3/wav，最多 3 个，总时长不超过 15 秒 |
| 混合素材 | 图片、视频、音频合计最多 12 个 |
| 生成时长 | 4-15 秒 |
| 输出 | 支持 2K |
| 引用 | `@图片1`-`@图片9`、`@视频1`-`@视频3`、`@音频1`-`@音频3` |

注意：
- 不要建议上传写实真人脸部素材，Seedance 可能拦截。
- 有参考视频时生成成本更高。
- 做视频延长时，生成长度写“新增部分”的时长。

## 执行流程

### 1. 判断任务类型

- 已有剧本/台词/分场文本：默认主流程。保留台词和剧情，拆成 Seedance 片段，输出可复制提示词。
- 单个场景或镜头：直接产出一个 4-15 秒 Seedance 提示词。
- 已有提示词优化：诊断问题，给出修正版和原因。
- 多集系列：先拆集，再为每集生成可独立生产、可连续拼接的提示词。
- 完整小说、故事、大纲：仅当用户明确要求“帮我写剧本/改编故事”时，才执行故事改编流程。

如果输入缺少关键信息，最多问 3 个问题；能合理默认时直接继续。默认值：
- 时长：15 秒
- 画幅：9:16 竖屏
- 风格：电影写实，除非题材明显更适合水墨、国漫、动漫、科幻或商业摄影
- 字幕/配乐：默认无字幕、无 BGM、无配乐

### 2. 确认制作参数

需要明确或自动推断：
- 视觉风格：写实、动画、水墨、科幻、复古、电影感、商业摄影等。
- 集数和时长：如 5 集 x 15 秒、20 集 x 15 秒、单条 8 秒。
- 画幅比例：9:16、16:9、2.35:1。
- 情绪基调：史诗、温馨、悬疑、欢快、忧伤、爽感等。
- 核心卖点：绝境反杀、复仇爽剧、治愈温馨、悬疑惊悚、产品亮点等。
- 可用素材：图片、视频、音频分别是什么，用途是什么。

### 3. 读取剧本并拆成 15 秒片段

处理已有剧本时，先建立“剧本保真表”：

- 角色：A/B/其他角色分别是谁。
- 场景：地点、时间、站位关系、关键道具。
- 原始台词：哪些句子必须原样保留。
- OS/VO：哪些旁白和内心音必须保留在时间轴，并匹配人物反应。
- 后期字幕标记：仅作为时间/信息提示，不生成画面文字。
- 动作：哪些动作是剧情必须发生的。
- 分场标记：识别 `第X集`、`X-X`、`日/夜`、`内/外`、`人物`。
- 情绪转折：每段只承载一个主要变化。
- 上下文：上一段怎么结束，下一段怎么开始。

15 秒拆段规则：

- 对话段：按台词长度动态判断；可为一句长台词拆分 + 他人反应，或 2-4 句短台词 + 1-2 个听者反应。
- 情绪段：只完成一个情绪变化点。
- 动作段：只完成一个连续动作链。
- 冲突段：只完成一个冲突升级点。
- 转场段：只完成一个空间或状态变化。

如果一段包含过多台词、多个反转、回忆、动作和转场，必须拆开，不要压进同一个 15 秒。

拆段判断：
- 必须拆：长解释+对方回应、对话+肢体冲突+揭示、多人同时争吵、跨地点/跨时间、闪回+现实动作。
- 可同段：一句长台词自然断开+一个反应、2-4句短对白、对白+同一情绪点OS、一个连续动作链。
- 不确定时先提醒，再给拆分建议，不要静默压缩。

对话戏时间轴优先使用：建立站位 -> 说话者完整说一句 -> 听者反应 -> 回应或留钩子。长台词只在标点后拆，OS 优先安排角色本人反应镜头。

动作戏优先写连续动作线：起始状态 -> 发力/移动方向 -> 接触或变化 -> 结果状态 -> 下一段可承接状态。可润色动作连续性，不可改变剧情结果。

### 4. 按戏型选择写法

| 戏型 | 写法 |
|---|---|
| 对话戏 | 分镜式：谁说话拍谁，台词/OS/VO都留在时间轴，句间插听者反应，保持站位 |
| 动作戏 | 剧情线式：可适度扩写动作起点、方向、终点，让模型保持动作连贯 |
| 情绪戏 | 克制反应：写反应方向、视线、看向镜头 |
| 长对话 | 跨段承接：上一段 A 结尾，下一段 B 开口回应 |
| 群戏 | 控制同框人数，明确主次，减少角色混乱 |

### 5. 生成剧本结构

完整改编时，先输出故事分析，再输出分镜头剧本。

仅在用户明确要求从故事/大纲写剧本时使用本节；已有剧本转提示词时跳过本节。

```markdown
# [标题] - 剧本

一、核心梗
[2-4字短语，如：绝境反杀、复仇爽剧]

二、故事梗概
故事背景：[时间、地点、初始状态]
开场冲突：[什么打破现状]
主角画像：[核心特质、当前状态]
主线事件：[推进 -> 高潮 -> 解决]
结局：[最终结果、角色转变]

三、一句话卖点
[情绪号召 + 核心冲突 + 视觉高潮]

四、人物小传
**角色名**
视觉形象：[年龄、外貌、服装、标志性道具]
身份背景：[社会身份、与主角关系、过往经历]
核心标签：[2-4个特质]
性格特点：[正常状态 -> 压力下表现 -> 爆发时]
金句：[最能代表角色的台词]

五、剧本大纲
前期（起）：[建立世界、主角现状、触发事件]
中期（承/转）：[冲突发展、转折点]
后期（高潮）：[最终对决、情感爆发]
尾声（收尾）：[结局状态、主题升华、余韵]

六、剧本正文
```

### 6. 写分镜头剧本

每个镜头必须以 `△ ` 开头。

```markdown
第X集
X-X [日/夜] [内/外] [场景名称]
道具：[道具列表]
出场人物：[角色列表]

△ 【空镜/开场镜头】[景别 + 运镜 + 画面 + 光影 + 氛围]
△ [镜头2：景别 + 运镜 + 动作 + 细节]
角色名（os）：[内心独白/旁白]
△ 【反应镜头】[无台词反应动作]
角色名（情绪）：[对白]
△ [连续动作 A -> B -> C]
【闪回】[回忆镜头]
【闪回结束】
△ [回到现实的情绪反应]
角色名（vo）：[画外音，人物不在画面]
△ 【空镜】[结尾氛围镜头]
【后期字幕标记：无；画面不生成文字】
```

镜头语言必须具体：
- 景别：远景、全景、中景、近景、特写、大特写。
- 运镜：推、拉、摇、移、跟、环绕、升降、希区柯克变焦、一镜到底、手持晃动。
- 角度：平视、俯拍、仰拍、低角度、鸟瞰、第一人称视角。
- 节奏：慢动作、快切、延时、升格、硬切、卡点、定格慢放。
- 焦点：浅景深、深景深、焦点转移、虚化背景。

对白规则：
- `角色名（os）`：内心独白或旁白。
- `角色名（vo）`：画外音，人物不在画面。
- `角色名（惊/怒/喜/怯/冷笑）`：带情绪的对白。
- 长台词要在自然句号、问号、感叹号后拆句，句间插入无台词反应镜头。
- 不要在一句话说到一半时切镜头。
- 谁说话就给谁镜头；听者反应镜头必须无台词。

15 秒镜头密度：
- 对话/情感：5-6 个镜头，每镜 2.5-3.5 秒。
- 动作/战斗：5-7 个镜头，每镜 2-3 秒。
- 蒙太奇：6-8 个镜头，每镜约 2 秒。

### 7. 创建素材清单

为所有视觉资产编号，后续分镜只使用编号引用。

| 类别 | 编号 | 用途 |
|---|---|---|
| 角色 | C01-C99 | 主角、配角、群众角色 |
| 场景 | S01-S99 | 关键地点、外景、室内空间 |
| 道具 | P01-P99 | 关键物件、武器、产品、线索 |

素材提示词格式：

```markdown
### [编号] - [名称]
[统一风格前缀]，[详细视觉描述]，[技术规格]
```

风格必须匹配题材：
- 仙侠/国风幻想：3D 国漫、东方幻想概念设计。
- 古风/历史：水墨、工笔、电影写实古装。
- 科幻：CG 概念设计、赛博朋克、硬科幻工业设计。
- 现实短剧：电影写实摄影。
- 美食/产品：商业摄影、干净布光、细节展示。
- 动漫：对应动画或赛璐璐风格。

### 8. 生成 Seedance 提示词

优先使用两种格式之一。

结构化标签式，适合单个场景、重氛围、重画质：

```markdown
【风格】[风格]，极致画质，高对比度光影，[镜头风格]，[时间]
【画幅】竖屏 9:16
【镜头】ARRI MINI拍摄，中焦段 [焦段]，适中光圈 [光圈]，电影写实人像
【环境】[场景详细描述]
【布光】[主光源 + 方向 + 补光]
【光影】侧顺光为主，环境反射补亮暗部，保留面部自然阴影层次
【色调】[色温]，[对比度]，画面干净清爽
【特效】背景轻微虚化散景，轻微胶片颗粒感，保留皮肤细节与毛孔质感
【运镜】手持拍摄，自然呼吸感运镜，[具体运动]
【构图】非对称构图，禁止越轴，镜头语言符合现实逻辑
【表演】动作自然流畅，加强眉眼鼻表演，每个镜头人物必须有眨眼动作，不生成多个相同角色
【角色】[role chips]
【站位】[整体场景站位描述：谁在哪里、面朝谁、间距、视线连线]
【道具】[prop chips，如有]

剧情动作线：
[按时间顺序描述具体动作]

反相约束：
禁止越轴；不生成多个相同角色；每个镜头人物必须有眨眼动作；全片不生成任何文字；无BGM、无配乐、无字幕。
```

时间轴式，适合 15 秒完整叙事：

```markdown
[风格描述]，15秒，9:16竖屏，[整体氛围]

0-3秒：[场景建立] - [运镜]，[画面描述]，[氛围]
3-7秒：[主体引入/发展] - [运镜]，[动作]，[情绪]
7-12秒：[高潮/冲突] - [运镜]，[情绪爆发]，[视觉冲击]
12-15秒：[收尾/落版] - [运镜]，[最终画面]，[余韵]

【声音】仅环境音：[具体环境音]；对白/OS/VO保留在时间轴；无BGM、无配乐
【参考】@图片1 [用途]，@图片2 [用途]
【反相】禁止越轴；不生成多个相同角色；每个镜头人物必须有眨眼动作；全片不生成任何文字；无字幕。
```

@引用规则：
- `@图片1 作为首帧`
- `@图片2 作为角色参考`
- `@图片3-5 作为场景参考`
- `@视频1 参考运镜`
- `@视频2 参考动作节奏`
- `@音频1 用于环境音`
- `@音频2 用于对白音色`
- 多素材时必须先列上传位置，避免引用错位。

视频延长写法：
- 后续段：`将@视频1延长15s。[新增时间轴内容]`
- 向前延长：`向前延长Xs，[前置剧情时间轴]`
- 衔接要求：上一段 `12-15秒` 的画面状态必须与下一段 `0-3秒` 完全一致。

## 提示词写法规则

- 写动作，不写心理：把“害怕”改成“瞳孔微缩，肩膀向后缩，呼吸变浅”。
- 微表情拆成显性动作：把“压抑狂喜”改成“嘴角短暂上扬即收，喉结微动”。
- 连续事件用 `->` 串起来，确保动作不可拆散。
- 用速度副词替代情绪词：猛然、瞬间、快速、缓慢、轻轻。
- 单人情绪反应镜头，视线必须落向镜头，建立角色与观众的直接情感连接。
   - ❌ "视线落向前方雪面"、"偏头看向身旁"
   - ✅ "视线怯怯落向镜头"
- 每个关键动作前重复当前身体状态，防止模型自动改姿势，如“躺在地上的主角”。
- 夜晚场景必须写具体夜间光源：月光、台灯、烛光、路灯、霓虹，不要写会暗示白天的自然光。

## 最终检查

交付前逐项检查：

- 每个镜头以 `△ ` 开头。
- 对应原剧本范围明确。
- 原场号已保留。
- 本段戏剧任务明确，15 秒没有塞过多内容。
- 如果内容过密，已主动拆段并说明原因。
- OS/VO 已保留在时间轴并匹配人物反应；后期字幕标记没有变成画面字幕。
- 运镜、景别、角度、节奏、焦点写具体。
- 台词标注正确，os/vo/情绪不混用。
- 谁说话就拍谁；反应镜头无台词。
- 长台词只在自然句读后拆分，不句中切镜头。
- 对话戏使用分镜，动作戏使用连续剧情线。
- 每集有情绪弧线：开场 -> 发展 -> 高潮 -> 释放。
- 时间轴覆盖完整 4-15 秒，无时间空洞或重叠。
- 所有 `@图片X/@视频X/@音频X` 在上传列表里能对应到素材。
- 角色、场景、道具编号不混用。
- 11 个技术标签齐全。
- 每个片段都有完整技术标签，没有依赖上一片段。
- 反相约束包含：禁止越轴、不生成多个相同角色、每个镜头人物必须有眨眼动作、全片不生成任何文字、无BGM/无配乐/无字幕。
- 多段视频的上一段结尾状态与下一段开场状态一致。
- 原始剧本台词没有丢失、重复或跨段重复。
- 用 `references/quality-rubric.md` 做最终质量判断；任一硬性门槛失败就重写该片段。
- 质量目标是结构、节奏、保真、技术标签和连续性先达标；不要为了追求复杂表演而把提示词写满。

## 常见问题处理

| 问题 | 修正 |
|---|---|
| 夜晚变白天 | 改成具体夜间光源，加“冷夜色基调” |
| 餐厅变太暗 | 写“暖色环境光均匀铺满，金色装饰反射光，水晶吊灯散光” |
| 台词太急 | 加“语速自然平缓，断句分明，句尾有明显停顿” |
| 抢夺没拉扯感 | 写“死死攥住往回拽、身体被拖得前倾、布料绷紧抖动” |
| 松口气出不来 | 写“肩膀塌下 -> 闭眼深呼吸 -> 攥拳的手松开” |
| 多人对话僵硬 | 每 4-6 秒插入听者的无台词反应镜头 |
| 角色姿势丢失 | 每个关键动作前重复“坐着/躺着/跪着/背对镜头”等状态 |
| 出现相同角色克隆 | 检查是否遗漏“不生成多个相同角色”，减少同画面相似人物 |
