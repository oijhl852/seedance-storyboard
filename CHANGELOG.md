## 2.4.0 - 2026-07-15

对应提交：[3100dbb](https://github.com/oijhl852/agent-skill-sync/commit/3100dbb837fda2b51de1d09e81d4159dc6fb5e06)

### 迭代重点

从“仓库里散落着主 skill 和多个独立子 skill”转变为“只安装一个目录即可使用全部能力”。

### 变化

- 将 13 个子 skill 移入 `awesome-seedance/references/subskills/`，由主 skill 按需读取。
- 将社区语料、精选案例和辅助脚本一起收进 `awesome-seedance` 包。
- 新增 `agents/openai.yaml`，补齐 Codex skill 的界面和触发配置。
- 将主 skill metadata 调整为官方校验能够识别的结构。
- 更新搜索、语料构建和验证脚本中的路径。
- 将示例和剧本参考统一归入 `references/`。

### 清理

- 删除仓库中的旧版 `backup/`。
- 删除废弃的 `seedance-browser-injector/` 入口。
- 删除重复的 `docs/`、`structured-prompt.md` 和 skill 内 README。

### 验证

- 自检覆盖 1 个主包、13 个子模块和 4776 条社区语料。
- 通过官方 `quick_validate.py` 格式校验。

## 2.3.0 - 2026-07-15

对应提交：[ca02aa1](https://github.com/oijhl852/agent-skill-sync/commit/ca02aa1)

### 迭代重点

把实际制作时容易遗漏的流程写成固定步骤，并把社区提示词从“零散参考”升级为可以本地检索和评分的语料库。

### 变化

- 固化按集准备输入、场景基础提示词锁定、戏剧节拍拆段、逐段检查、资产对应和安全注入流程。
- 明确最终提示词必须用自然语言描述变化，不使用箭头或符号代替过程。
- 增加社区语料搜索、构建和完整性验证脚本。
- 整合 4776 条完整 CSV 提示词和 106 条 Markdown 快速预览。
- 增加社区案例质量评分规则，区分可参考结构和低分反例。
- 强化凭证保护、空白片段确认、失败停止和刷新验证。

## 2.1.0 - 2026-07-15

对应提交：[299a209](https://github.com/oijhl852/agent-skill-sync/commit/299a209)

### 迭代重点

参考 Superpowers 的职责拆分方式，把一个大型单文件 skill 改造成主编排与专项子 skill 配合的结构。

### 变化

- 建立 1 个主 skill + 13 个子 skill 的模块体系。
- 将剧本读取、拆段、对白、动作、故事改编、资产、提示词、质量和面板注入分别交给独立模块。
- 主 skill 负责判断用户输入、选择子模块和传递阶段结果。
- 保留旧版单文件 skill 作为当时的迁移备份。
