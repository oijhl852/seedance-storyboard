# Seedance 提示词框架摘要

来源：[songguoxs/seedance-prompt-skill](https://github.com/songguoxs/seedance-prompt-skill)，MIT License。
本文件是独立整理的参考摘要，不加载上游 skill 的角色定义、触发规则或执行指令。

## 核心输入能力

- 文本：自然语言描述。
- 图片：角色、产品、场景、首帧和尾帧参考。
- 视频：运镜、动作、节奏、特效和延长参考。
- 音频：音色、对白、节奏和背景声音参考。

平台参数可能变化，实际使用前需在当前页面确认。上游资料记录的范围包括：4-15 秒生成、最多 9 张图片、最多 3 个视频、最多 3 个音频，以及图片/视频/音频合计最多 12 个文件。

## 十类提示词任务

1. 纯文本生成：主体 + 动作序列 + 环境光影 + 镜头语言 + 风格。
2. 一致性控制：明确每张参考图对应的角色、产品或场景。
3. 运镜与动作复刻：分别说明参考视频中的运镜、动作或节奏用途。
4. 特效与创意复刻：指出保留的形式和需要替换的主体。
5. 剧情创作：以分镜或图片为基础，明确允许模型补全的范围。
6. 视频延长：上一段尾帧状态必须等于新增段首帧状态。
7. 声音控制：对白、音色、环境音和BGM分别说明来源。
8. 一镜到底：明确不中断的运动路线、遮挡和场景过渡。
9. 视频编辑：列出保留项、替换项和禁止变化项。
10. 音乐卡点：把动作、剪辑和镜头切换绑定到节拍点。

## @引用原则

- 引用不能只写 `@图片1`，必须说明它是角色、场景、首帧还是材质参考。
- 多个素材按出现顺序编号，并在生成前检查编号与上传顺序一致。
- 同一素材不要同时承担互相冲突的用途。
- 资产 ID 和网页 chip 仍以 `sd-chip` 的真实页面数据为准。

## 长视频策略

超过 15 秒时拆成多个片段。每段都必须记录：当前起始状态、片段目标、结束状态、下一段衔接点。视频延长的时长指新增部分，不是最终总时长。

## 使用边界

本摘要只提供能力分类和提示词结构。时长、镜头密度、台词保真、资产安全和质量阻断规则，以当前项目的 `sd-segment-split`、`sd-prompt`、`sd-chip` 和 `sd-quality` 为准。
