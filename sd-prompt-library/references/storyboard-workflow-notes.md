# Storyboard Generator 方法摘要

来源：[liangdabiao/Seedance2-Storyboard-Generator](https://github.com/liangdabiao/Seedance2-Storyboard-Generator)。
上游仓库未发现明确许可证，因此本文件不复制其完整原文，只记录独立整理的方法和检查重点。

## 三层交付物

1. 剧本：人物、场景、台词、动作和剧情结果。
2. 素材清单：角色、场景、道具的稳定编号和视觉描述。
3. 分镜：按集和片段把剧本转换成可执行的镜头、时间轴和提示词。

三层之间必须能追溯：分镜中的角色和道具应来自素材清单，分镜中的台词和动作应来自剧本。

## 多集工作流

- 每集先定义唯一戏剧任务，再决定镜头数量。
- 上一集尾帧与下一集首帧记录人物位置、朝向、身体状态、时间和光线。
- 角色服装、发型和关键道具跨集保持稳定，除非剧本明确发生变化。
- 动作结果不能跨集消失，例如上一集倒地，下一集不能无说明直接站立。

## 适合 Agent 学习的案例类型

- 完整的“剧本 → 素材清单 → 多集分镜”项目。
- 对话戏中的说话者镜头和听者反应。
- 动作戏中的起始状态、受力方向和结果状态。
- 历史、武侠、战争等多人场景的资产一致性。

## 使用边界

这些方法用于检查结构和连续性，不代表上游每个案例都适合直接复用。实际输出仍以 `sd-segment-split`、`sd-dialogue`、`sd-action` 和 `sd-quality` 的当前规则为准。
